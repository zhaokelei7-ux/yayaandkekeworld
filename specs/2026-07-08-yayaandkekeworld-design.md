# Yaya & Keke World: 3D 像素方块网页应用 — 设计文档

> 一个类似 Minecraft 风格的 3D 像素方块世界，使用 Three.js + Vite 构建，支持 PC 与移动端。

---

## 1. 技术选型

| 层 | 选择 | 理由 |
|---|---|---|
| 语言 | JavaScript (ES Module) | 零类型编译，开发迭代快 |
| 3D 引擎 | Three.js r160 | 成熟 WebGL 封装，社区活跃 |
| 构建工具 | Vite 5 | 极速 HMR，零配置上手 |
| 物理 | 自研 AABB 碰撞系统 | 比 cannon.js 轻量百倍，方块世界专用 |
| 纹理 | Canvas API 程序化生成 | 零外部图片资源，自包含部署 |
| 包管理 | npm + package.json | 标准流程 |

## 3. 架构与数据流

### 3.1 游戏主循环

```
requestAnimationFrame
  │
  ├─ Controls.update(dt)       → 更新俯仰/偏航角，产出移动方向向量
  ├─ Player.update(dt, input)  → 应用移动方向 + 物理更新
  │     └─ Physics.update(dt)  → 重力 += 9.8*dt, 三轴分别移动 + 碰撞回退
  ├─ World.update(playerPos)   → 检查区块是否需要加载/卸载
  ├─ Interaction.update()      → 射线检测，处理点击事件
  └─ renderer.render(scene, camera)
```

### 3.2 核心模块职责

| 模块 | 职责 | 不负责 |
|---|---|---|
| **Game** | 启动、循环调度、输入桥接 | 任何具体游戏逻辑 |
| **World** | 区块字典 CRUD、位置到区块映射 | 网格生成细节 |
| **Chunk** | 存储 16×16×h 方块 ID，构建/重建 Mesh | 方块的视觉外观 |
| **BlockTypes** | 定义方块颜色、名称、纹理创建 | 场景中的实例 |
| **Player** | 位置、速度、碰撞箱、跳跃 | 渲染、输入原始数据 |
| **Controls** | 将鼠标/触控 → 视角方向 + 移动意图 | 物理、位置计算 |
| **Physics** | 重力积分、AABB vs 方块碰撞响应 | 玩家状态管理 |
| **Interaction** | Raycaster 命中检测、方块放置/破坏 | 区块网格重建（通知 World 触发） |

## 4. 方块系统

### 4.1 方块类型

| ID | 名称 | 颜色 (Hex) | 生成位置 |
|---|---|---|---|
| 0 | AIR | — | 无方块 |
| 1 | GRASS | #4CAF50 (顶) / #8B5E3C (侧) | 地表最上层 |
| 2 | DIRT | #8B5E3C | 地表下 1-3 层 |
| 3 | STONE | #808080 | 地下 4+ 层 |
| 4 | SAND | #F4E4A0 | 中心平台 |
| 5 | COZE_RED | #E53935 | COZE 文字墙 |
| 6 | COZE_WHITE | #ECEFF1 | 文字墙背景 |

### 4.2 纹理生成

使用 Canvas API 为每种方块生成 16×16 像素纹理：
- 每个纹理绘制 2×2 像素格（模拟像素风格）
- 草地：顶面绿色，侧面棕色+绿色顶部条
- 沙地：米黄色带噪点
- COZE 红：亮红色带暗红边线

## 5. 世界生成

### 5.1 地形

- 使用多层正弦波叠加生成高度图 (x, z) → height
- 默认世界高度 64 层
- 地表高度映射到 [4, 12] 范围
- 地表最上层 → GRASS
- 地表下 3 层 → DIRT
- 之下 → STONE
- Y=0 层 → 基岩（不可破坏）

### 5.2 中心沙质平台

- 以世界原点 (0,0) 为中心，范围 -10..10 的方形区域
- 高度固定为 y=5
- 表面层全部替换为 SAND

### 5.3 COZE 文字墙

- 位于沙质平台边缘 Z 负方向（玩家出生点在原点，面向 -Z，墙在 -Z 方向，玩家出生即正对）
- 每个字母 5×7 方块，间距 2 方块
- 红色方块 (COZE_RED) 拼出字母，背景白色方块 (COZE_WHITE)
- 坐标相对沙质平台边缘居中
- 底层 y 与沙质平台平齐

## 6. 玩家与视角控制

### 6.1 第一人称参数

| 参数 | 值 |
|---|---|
| 玩家碰撞箱 | 0.6 × 1.8 × 0.6 (宽×高×深) |
| 眼睛高度 (相机偏移) | 1.6 |
| 移动速度 | 4.0 方块/秒 |
| 跳跃初速度 | 5.0 方块/秒 |
| 重力加速度 | 9.8 方块/秒² |
| 交互距离 | 5 方块 |

### 6.2 视角控制

**PC 端：**
- 点击画布 → PointerLock API
- mousemove → camera.rotation 更新（yaw/pitch）
- 水平灵敏度 0.002，垂直 0.002
- 垂直旋转限制 ±89°

**移动端：**
- 单指拖动 → 视角旋转（触摸 delta → yaw/pitch）
- 左侧虚拟摇杆 → WASD 移动
- 右下按钮 → 跳跃
- 右下按钮① → 放置方块
- 右下按钮② → 破坏方块

### 6.3 键盘绑定

| 按键 | 动作 |
|---|---|
| WASD | 前后左右移动 |
| 空格 | 跳跃 |
| 左键 | 放置方块 |
| 右键 | 破坏方块 |
| 1-6 | 选择方块类型 |
| 滚轮 | 切换方块类型 |

## 7. 物理系统

### 7.1 重力

```
每帧: velocity.y -= GRAVITY * dt
在 ground 接触时: velocity.y = max(velocity.y, 0), 置 grounded=true
```

### 7.2 AABB 碰撞检测

- 对 X/Y/Z 三轴分别处理（分离轴定理简化版）
- 每轴：移动 → 检测玩家 AABB 与周围方块 AABB 是否重叠 → 穿透修正
- 检测范围：玩家所在位置 ±2 区块内方块
- 修正方式：将玩家位置推到方块表面

### 7.3 跳跃

- grounded = true 时按下空格 → velocity.y = JUMP_SPEED
- 空中不可再次跳跃

## 8. 区块系统与性能优化

### 8.1 区块规格

| 参数 | 值 |
|---|---|
| 区块尺寸 | 16 × 16 水平，高度 64 |
| 渲染距离 (PC) | 6 区块半径 |
| 渲染距离 (移动端) | 4 区块半径 |
| 最大加载区块数 (PC) | ~169 (13 × 13 网格) |
| 最大加载区块数 (移动端) | ~81 (9 × 9 网格) |

### 8.2 区块生命周期

```
UNLOADED → [玩家移入距离] → LOADING (生成地形) → LOADED (构建网格) 
→ RENDERED (加入场景) → [玩家移出距离 + 超时] → UNLOADED (移除场景 + 释放)
```

### 8.3 面剔除

每个方块检查 6 个邻面——若邻接方块存在且非透明 → 该面不生成三角形

### 8.4 其他优化

- 渲染距离外区块自动卸载
- 当页面不可见时暂停 RAF 循环 (Page Visibility API)
- 设备像素比上限 2
- 方块网格复用 BufferGeometry，减少 GC

## 9. 移动端适配

- CSS `100vw` / `100dvh` 确保全屏
- 虚拟摇杆：Canvas 2D 绘制，直径 120px
- 触控按钮：至少 44×44px（WCAG 触控目标）
- 响应式断点：
  - > 1024px → PC 布局 + PointerLock
  - 768-1024px → 平板，减少渲染距离
  - < 768px → 手机，虚拟摇杆 + 更小 HUD
- 横屏推荐，竖屏提示旋转设备

## 10. 交互系统

### 10.1 Raycaster

- 使用 Three.js `Raycaster`，从相机发射
- 检测 `intersectObjects` 命中所有区块网格
- 命中距离 ≤ 5 方块 → 有效

### 10.2 放置

- 从命中面的法线方向偏移一个方块位置
- 检查新方块不与玩家碰撞箱重叠
- 默认放置 SAND，可通过数字键切换

### 10.3 破坏

- 命中方块 ID ≠ 基岩 → 标记移除
- 通知 World 重建对应区块的网格

---

## 11. 错误处理

- WebGL 不支持时显示降级提示（`index.html` 内联）
- 缺失区块数据返回默认值而非崩溃
- 所有循环/更新函数含边界检查
